

debug错误调试
```
即程序中出现的错误，这个错误分为语法错误和逻辑错误现代编译器具有语法提示和检测的功能，所以语法错误在代码
编写阶段就可以清晰地发现，而逻辑错误很难查发现，所以对于错误排查主要针对的是逻辑错误，这个过程也称为调试
```
---

传统的调试方法
```
通过人眼观察和分析程序，以及关键位置进行log日志的输出，但这种方
式在某些情况下很难找出程序的错误，于是出现了更专业的程序调试工具
```

图形化调试工具
```
例如visual studio集成开发环境自带的图形化调试组件工具
```

命令行调试工具
```
GDB(GNU DeBugger)：GNU工具链的一个调试器，专门用于程序的调试
```
---

GDB常用命令


启动与停止
```
带调试信息编译         gcc/g++ -g ...
启动调试              gdb xxx(注:带调试信息的程序才能被调试)
退出调试              quit/q
```

程序运行与控制
```
run/r                运行
start                运行并在main前停下
continue/c           继续运行
next/n               单过程执行(不进入函数)
step/s               单语句执行(进入函数)
finish               执行直到函数返回前
```

断点
```
break/b 行号/函数名    设置断点
break 行号 if 条件     设置条件断点
info breakpoints      查看断点信息
delete 编号            清除某个断点
clear                 清除所有断点
```

内存与变量
```
print/p 变量名        打印变量值
info regisiter       查看寄存器信息
watch 变量            当该变量被修改时暂停

x/10x 地址             查看地址的内存(以16进制显示10个单位)
backtrace/bt          查看函数调用栈
frame 编号             切换到指定栈帧
info locals           查看当前栈帧的局部变量
```
