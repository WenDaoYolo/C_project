

CPU与核心
```
CPU是将核心、寄存器、高速缓存等相关硬件封装成整体后的一个统称，而核心是CPU中最重要的
部分，负责计算和处理数据，CPU内若只有单个核心则称为单核CPU，有多个核心则称为多核CPU
```

程序的执行
```
程序本身是静态的，运行起来后称为进程，程序的运行需要为其分配内存和处理核心等资源，程序从外存加载到内存，然后CPU核心执行程序任务
```

并发与并行
```
单个核心同一时刻只能处理单个任务，单个核心若要处理多任务，操作系统则会为每个任务划分时间片，核心按照时间片交
替执行不同任务，因为CPU核心速度极快，所以宏观上看起来好像是同时执行所有任务，而像这执行任务的方式则称为并发

并行则是真正意义上的同时进行，现代CPU采用多核技术，会为每个任务单独分配一个核心执行程序任务，但采用多核技术并不意味着一定能
实现并行，因为CPU核数有限 ，当任务数量多于核心时，操作系统为了能执行所有任务指令，会有部分核心仍然以并发的方式去执行多个任务
因为任务是实时变化的，这个过程中操作系统会根据不同场景采用不同的调度算法，动态调整核心资源的分配，这也是操作系统主要职责之一
```

任务池与任务调度器
```
任务池用于记录需要执行的任务，每当任务被创建后都会放入任务池，执行完毕后从任务池中移除
任务调度器是操作系统调度任务和分配核心的重要组件，它负责从任务池中取任务并为其分配核心
```

进程与线程
```
进程是运行的程序，是基本的资源分配单位，OS以进程为单位分配内存、CPU核心等资源
线程是进程的分支，是基本的任务调度单位，CPU核心以线程为单位执行和处理任务指令
```

用户态与核心态
```
是CPU核心的两种执行权限，用户态下只能执行用户级指令，核心态下可以执行所有指令
```

用户空间与内核空间
```
通过虚拟内存技术，将虚拟空间划分为用户空间和内核空间，分别映射到不同物理空间
内核空间可以使用和访问任意内存空间的数据，而用户空间则只能使用自己的内存空间
为应用进程分配的内存是用户空间，存储系统数据、硬件等重要数据的内存是内核空间
其中用户空间中的程序和指令运行在用户态，而内核空间中的程序和指令运行在核心态

这样划分后，进程只能使用自己的内存，不能访问内核空间，保证进程不能破坏系
统，进程之间也进行了安全隔离不能直接相互访问，保证进程之间也不能互相干扰
```

用户态与内核态的切换
```
内核态可以主动切换为用户态，但用户态不能主动切换为内核态，只有触发特定的条件才能进行切换
1.系统调用，用户程序访问核心资源时，通过系统提供的接口请求状态的切换，如文件读写、线程创建等
2.异常和中断，硬件发出中断(例如网卡硬盘等完成数据的读写)或用户程序出现错误(如内存越界、除0)
3.任务切换，当操作系统需要进程上下文切换或调度时，会触发状态的切换，例如进程之间需要进行通信
```

进程通信与线程通信
```
进程通信成本高，进程之间进行了隔离，物理空间不同不能直接相互的读写内存
所以只能通过OS重新调度和分配任务并且CPU核心状态切换后才能访问其他进程

线程通信成本低，线程在同一物理内存空间，线程之间可以直接读写内存，不需
要额外的内核介入和状态切换，共享进程资源的同时还拥有自己的栈和其他资源

所以这也是操作系统为什么这么设计，进程是基本的资源分配单位，线程才是基本的任务调度单位
进程不在同一空间，资源进行了隔离，更加安全，线程在同一空间，通信无需额外开销，更加高效
```

用户态与和核心态切换的流程分析
```
例如系统调用
1.处理核心接收到相关指令，保存上下文(寄存器值、PC、程序地址等信息)，再切换为核心态
2.处理核心对系统调用进行错误检测，调用无误后跳转到内核空间，执行系统调用的程序和指令
3.执行完成后将结果和相关信息写入用户空间，然后通过保存的上下文进行恢复并切换为用户态

其中状态切换的主要开销来自于上下文保存与切换、以及错误检测，因此进程通信成本高，频
繁通信要不断切换状态，OS需要持续进行处理，CPU核心也将大部分时间花在切换而非计算上
```
